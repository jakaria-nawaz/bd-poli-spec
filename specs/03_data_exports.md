# Spec: Data Exports and Sample Data

## Goal
Produce two kinds of output from the raw survey data:

1. **Private full exports** — complete anonymised datasets in `private_analysis/`
   for internal use, journalism, and sharing with trusted collaborators.
   These are gitignored and never committed.

2. **Public sample data** — a small, safe subset in `sample_data/`
   that is committed to the repository so anyone can clone and explore
   the structure without accessing the full dataset.

---

## Privacy Rules

The following fields must NEVER appear in any export file:

| Field | Reason |
|---|---|
| `emailOrFacebook` | Direct PII — contact identifier |
| `meta_ip` | Direct PII — IP address |
| `meta_isp` | Near-identifying |
| `meta_city` | Near-identifying |
| `meta_region` | Near-identifying |
| `meta_country` | Near-identifying (also ~94% null) |
| `meta_userAgent` | Device fingerprinting |
| `doc_id` | Firestore document ID — internal reference |
| `consentAccepted` | Not analytically useful |
| `q_101` through `q_105` | Demographic duplicate questions — already captured as flat fields |

The following fields ARE safe to include in all exports:

`age`, `gender`, `education`, `occupation`, `probashi`, `createdAt`,
`meta_platform`, `meta_language`, `q_1` through `q_22`,
`economicScore`, `socialScore`, `resultLabel`

## Script 1: `generate_export_files.py`

**Reads from:** `data/responses.parquet`
**Writes to:** `private_analysis/` (gitignored)

Produces two export sets:

### Set 1 — Anonymised (no scores, no political label)
For sharing with researchers who should derive their own scores.

Columns: `age`, `gender`, `education`, `occupation`, `probashi`,
`createdAt`, `meta_platform`, `meta_language`, `q_1` through `q_22`
(sorted numerically)

Output files:
- `private_analysis/survey_anonymised.csv`
- `private_analysis/survey_anonymised.parquet`

### Set 2 — With scores and label
For journalists and non-technical collaborators who need the pre-computed result.

Columns: same as Set 1, plus `economicScore`, `socialScore`, `resultLabel`

Output files:
- `private_analysis/survey_with_scores.csv`
- `private_analysis/survey_with_scores.parquet`

### Code requirements
- Sort `q_1` through `q_22` numerically (not lexicographically — `q_2` before `q_10`)
- Create `private_analysis/` with `os.makedirs(..., exist_ok=True)` before writing
- Print shape and column list after each save for verification

## Script 2: `generate_questions_reference.py`

**Reads from:** hardcoded question data (no file dependency)
**Writes to:** `sample_data/` (committed to git)

Produces a reference table mapping question numbers to Bengali and English text.
This file has no personal data and is safe to commit.

### Columns
| Column | Type | Description |
|---|---|---|
| `question_id` | int | Question number (1–22) |
| `category` | string | `"economic"` or `"social"` |
| `original` | string | Full Bengali statement |
| `english` | string | Full English translation |

### Join key
`question_id = N` maps to column `q_N` in any survey export file.

### Output files
- `sample_data/questions_reference.csv` (UTF-8 with BOM so Bengali renders in Excel)
- `sample_data/questions_reference.parquet`

### Code requirements
- Create `sample_data/` with `os.makedirs(..., exist_ok=True)` before writing
- Use `encoding="utf-8-sig"` for the CSV so Excel opens Bengali text correctly

## `sample_data/` — Public Folder

This is the only data folder committed to the repository.
It gives contributors and researchers enough to understand the schema
and run test queries, without exposing the full dataset.

### Files in `sample_data/`

| File | Source | Rows | Notes |
|---|---|---|---|
| `questions_reference.csv` | `generate_questions_reference.py` | 22 | Complete — no PII |
| `questions_reference.parquet` | `generate_questions_reference.py` | 22 | Complete — no PII |
| `survey_with_scores_sample.csv` | generate manually (first 100 rows of Set 2) | 100 | Anonymised subset |
| `survey_with_scores_sample.parquet` | generate manually (first 100 rows of Set 2) | 100 | Anonymised subset |
| `findings.md` | copied from `private_analysis/findings.md` | — | Key findings from full dataset |
| `ABOUT.md` | hand-written | — | Explains what is and isn't in this folder |

### How to regenerate `sample_data/` files

```python
import os, pandas as pd, shutil

os.makedirs("sample_data", exist_ok=True)

# questions reference — generated by generate_questions_reference.py
# (run that script directly)

# 100-row sample
df = pd.read_parquet("data/responses.parquet")
q_cols = sorted(
    [c for c in df.columns if c.startswith("q_") and c[2:].isdigit() and 1 <= int(c[2:]) <= 22],
    key=lambda x: int(x[2:])
)
cols = (
    ["age", "gender", "education", "occupation", "probashi",
     "createdAt", "meta_platform", "meta_language"]
    + q_cols
    + ["economicScore", "socialScore", "resultLabel"]
)
df[cols].head(100).to_csv("sample_data/survey_with_scores_sample.csv", index=False)
df[cols].head(100).to_parquet("sample_data/survey_with_scores_sample.parquet", index=False)

# findings.md
shutil.copy("private_analysis/findings.md", "sample_data/findings.md")
```

### `ABOUT.md` requirements
Must explain clearly:
- This is a sample — not the full dataset
- How to join questions_reference to the survey file
- The Likert scale values (-2 to +2)
- What fields were excluded and why
- Where to find full findings (findings.md in same folder)
- The sample caveat (self-selected, 88.7% male, 69% students)

## `private_analysis/` — Private Output Folder (gitignored)

This folder holds all outputs generated by `deep_analysis.ipynb` and the export scripts.
It is never committed. Anyone with Firebase credentials can regenerate it by running
`fetch_data.py` and then `deep_analysis.ipynb`.

```
private_analysis/
  charts/                         <- 10 PNG charts at 150 dpi
  tables/                         <- 10 CSV tables
  findings.md                     <- Auto-generated narrative
  survey_anonymised.csv           <- 17,259 rows, no scores
  survey_anonymised.parquet
  survey_with_scores.csv          <- 17,259 rows + economicScore, socialScore, resultLabel
  survey_with_scores.parquet
  survey_analysis_for_non_tech.csv  <- Pre-aggregated stats by question and subgroup (672 rows)
```
